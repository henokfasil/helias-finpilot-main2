## File Storage — Ministry of Revenue Compliant

Uploaded files (receipts, invoices, PDFs) are stored permanently on disk with human-readable names:

```
uploads/
  helias-ai/
    2026/
      03/
        2026-03-25_tx42_Ethio_Telecom.jpg
        2026-03-23_tx38_Addis_Tech.pdf
      04/
        2026-04-01_tx55_CBE_Bank.jpg
```

- Files are **never deleted**, even if the transaction is soft-deleted
- Named: `{date}_tx{id}_{counterparty}.{ext}` after transaction confirmation
- Organized by `{company_slug}/{year}/{month}/`
- At year-end: send `/export_receipts 2026` → receive a ZIP file ready for MoR

## Transaction Lifecycle

```
User sends text or file
        ↓
Extraction Agent (OpenAI)
        ↓
Validation Agent (completeness check)
        ↓
Bot shows preview → asks yes / no / edit
        ↓
On "yes" → duplicate check → save as confirmed → rename receipt file → audit log
On "no"  → discard
On "edit" → apply field correction → re-show preview
```

## Issues Encountered & Fixed

1. **Python 3.9 incompatibility** — `str | None` syntax not supported in SQLAlchemy annotations → fixed with `Optional[str]` from `typing`
2. **Detached instance error** in seed script → fixed by reading `.name` and `.id` inside the session context
3. **Two bot instances conflicting** → killed both with `pkill -f app.main`, restarted single clean instance
4. **/delete command** added after initial build — allows soft-deleting confirmed transactions
5. **File storage reorganized** — from random UUID names in flat `uploads/` to organized `uploads/{slug}/{year}/{month}/` with human-readable filenames
6. **/receipts and /export_receipts** commands added for Ministry of Revenue compliance
