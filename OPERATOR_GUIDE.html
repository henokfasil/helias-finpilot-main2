How the Bot Works

1.1 — Sending a Transaction

The client types a message in plain language directly in Telegram. No forms, no app, no spreadsheet.

Client TypesBot Extracts
Paid 3,500 ETB to Ethio Telecom for internetExpense · 3,500 ETB · Ethio Telecom · Internet & Telecom
Received $400 from Addis Tech for consultingIncome · 400 USD · Addis Tech · Consulting
Bought 2 chairs, 4,800 birr cashExpense · 4,800 ETB · Equipment · Cash
(send a photo of a receipt)All fields extracted using AI vision

1.2 — Confirmation Flow

After every message or file, the bot shows a preview and waits for approval before saving anything.

Helias_FinPilot · bot
💸 Extracted Transaction

Type:         expense
Amount:       3,500 ETB
Date:         2026-03-25
Counterparty: Ethio Telecom
Description:  internet service
Category:     Internet & Telecom
Confidence:   100%

Is this correct? Reply:
  ✅ yes — save it
  ❌ no — discard it
  ✏️ edit — correct a field

The client replies yes, no, or edit. Nothing is saved until the client confirms.

1.3 — Supported File Types

Clients can send photos, PDFs, and screenshots directly in Telegram. The AI reads the content automatically.

File TypeExamples
PhotosReceipts, invoices, handwritten notes, bank slips
PDF documentsBank statements, formal invoices, contracts
ScreenshotsMobile payment confirmations, online receipts

Three easy steps: 1. Send /transactions — find the ID number (e.g. #12)
2. Send /delete 12 — removes it from reports
3. Re-type the correct transaction as a new message

Onboarding a New SME Client

1

Create a Telegram Bot for the Client

Open Telegram → search @BotFather → send /newbot → choose a name and username (must end in bot, e.g. acme_finance_bot) → copy the token.

Each client gets their own dedicated bot. Never share one bot between multiple companies.
2

Configure the .env File

Create a new .env file with the client's details:

TELEGRAM_BOT_TOKEN=<their bot token>
OPENAI_API_KEY=<your shared OpenAI key>
DATABASE_URL=sqlite:///./finpilot_acme.db
DEFAULT_COMPANY_NAME=Acme Trading PLC
DEFAULT_COMPANY_CURRENCY=ETB
DEFAULT_ADMIN_TELEGRAM_ID=<client's Telegram user ID>

To get the client's Telegram ID: ask them to message @userinfobot on Telegram — it replies with their ID number.

3

Install and Run

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python scripts/seed_data.py
nohup python -m app.main > finpilot.log 2>&1 &
4

Customise Categories (Optional)

Default categories are seeded automatically. For specialised businesses, categories can be adjusted before first run. Example for a restaurant: replace Cloud & API Costs with Food & Ingredients, Contractors with Kitchen Staff.

5

Test Before Handover

Verify these 5 things work before giving the bot to the client:

1.Send a text transaction → bot extracts correctly
2.Send a receipt photo → bot reads from image
3./summary → shows correct totals
4./monthly_report → generates report
5./delete 1 → removes test transaction
6

Hand Over to Client

Share the bot username with the client (e.g. @acme_finance_bot). They open it in Telegram and send /start. Done.

What to Tell Clients

What the bot CAN do

  • Record income and expenses from plain text or photos
  • Categorise transactions automatically using AI
  • Generate monthly and annual financial summaries
  • Export data as CSV for sharing with an accountant
  • Store all receipts and invoices permanently
  • Ask clarifying questions when something is unclear
  • Support ETB, USD, and EUR
  • Track every change with a full audit trail

What the bot CANNOT do (Phase 1)

  • Connect directly to a bank account
  • File taxes or calculate tax liability automatically
  • Handle payroll calculations
  • Support multiple users simultaneously (coming Phase 2)
  • Work without internet connection
  • Replace a certified accountant for official audit purposes

Data & Privacy

What clients must know: All transaction data is stored on a private server. Transaction text is sent to OpenAI for AI processing only — the full financial database stays on the private server. No data is shared with third parties other than OpenAI's extraction service.

Common Client Questions

What if I send the wrong amount?
Use /transactions to find the ID, then /delete [ID]. Re-send the correct message immediately after.
What if the bot misunderstands my message?
Reply no when it shows the preview. Then try again with more detail — include the exact amount, currency, and who you paid.
Can I use it in Amharic?
The AI understands some Amharic context, but English works more reliably in the current version. Full Amharic support is planned for the next version.
What currencies does it support?
ETB (Birr), USD, and EUR. Always mention the currency in your message if it is not ETB.
Can my accountant access the reports?
Use /export to download a CSV file, then share it by email or WhatsApp. A web dashboard for accountants is coming in the next version.
What happens if the internet goes down?
The bot is unavailable during outages. Messages sent during that time are queued by Telegram and processed automatically when the connection returns.
Is my financial data safe?
Yes. Data is stored on a private server. Only the transaction text is sent to the AI service for extraction — your full financial history never leaves the private server.

Operator Maintenance

Check if the bot is running

ps aux | grep "app.main" | grep -v grep

If you see a result with a process ID — it is running. If you only see the grep command itself — it has stopped.

Restart the bot

pkill -f "app.main"
cd /path/to/Financial_Reporting_AI
nohup venv/bin/python3 -m app.main > finpilot.log 2>&1 &

View logs

tail -f finpilot.log        # live stream
tail -100 finpilot.log      # last 100 lines
grep "ERROR" finpilot.log   # errors only

Back up client data

cp finpilot.db backups/finpilot_$(date +%Y-%m-%d).db
Recommendation: Set up a daily automatic backup using macOS cron or launchd. Contact the Helias tech team for the configuration file.
